# Mehrsprachigkeit – DE / EN
## Routing, Übersetzungen, Toggle-Logik

---

## Strategie

- **Standardsprache:** Deutsch (`/` oder `/de/`)
- **Englisch:** `/en/`
- **Umschalter:** Im Header, immer sichtbar ohne Scrollen
- **Kein Auto-Redirect:** Nutzer wählt explizit. Kein Redirect nach Browser-Sprache.
- **Fallback:** Fehlende EN-Übersetzung → DE-Inhalt anzeigen (kein leerer Content)

---

## Astro Routing

```
src/pages/
  index.astro          → redirect zu /de/
  de/
    index.astro        → DE Startseite
  en/
    index.astro        → EN Startseite
```

Alternativ mit Astro i18n Routing (ab Astro 4):
```js
// astro.config.mjs
export default defineConfig({
  i18n: {
    defaultLocale: 'de',
    locales: ['de', 'en'],
    routing: { prefixDefaultLocale: false }
  }
})
```

## Sprach-Toggle Komponente

```astro
---
// src/components/layout/LanguageToggle.astro
const { currentLang } = Astro.props
const otherLang = currentLang === 'de' ? 'en' : 'de'
const otherPath = Astro.url.pathname.replace(`/${currentLang}`, `/${otherLang}`)
---

<a
  href={otherPath}
  aria-label={currentLang === 'de' ? 'Switch to English' : 'Zur deutschen Version wechseln'}
  class="lang-toggle"
>
  {currentLang === 'de' ? 'EN' : 'DE'}
</a>
```

**Platzierung:** Header, rechts, immer sichtbar.
**Style:** Dezent – kein auffälliger Button. Textlink mit leichter Hervorhebung.

## SEO – hreflang & Canonical

In jedem Astro Layout:
```astro
<link rel="canonical" href={`https://digipub.de${Astro.url.pathname}`} />
<link rel="alternate" hreflang="de" href={`https://digipub.de/de${slug}`} />
<link rel="alternate" hreflang="en" href={`https://digipub.de/en${slug}`} />
<link rel="alternate" hreflang="x-default" href={`https://digipub.de${slug}`} />
```

## Redaktionelle Regeln für EN-Texte

- EN ist kein wörtliche Übersetzung – es ist eine eigenständige Stimme auf Englisch
- Tonalität: gleich (Du → You, direkt, peer-to-peer)
- "Mehrwert ist Marke" auf EN: "Value is Brand." (nicht "Added value is brand")
- "Strukturproblem" auf EN: "Structural problem" (nicht "structure problem")
- Alle heiligen Sätze (Peak + End) müssen einzeln übersetzt und freigegeben werden
